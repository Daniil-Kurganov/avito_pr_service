## Зпуск
Для запуска выполните `docker-compose up --build` в корне проекта. После успешного старта, API будет доступно по адресу `http://127.0.0.1:8080/...`.
## Вопросы и самостоятельно приянтые решения
### Логгирование
Логирование инициализируется на уровне "Debug" и выводится в консоль.
### API и валидация запросов
При реализации, опираюсь на описанное API и что такие ситуации как запрос на добавление команды с пустым составом не будет поставляться на вход. Иначе, сервер будет отправлять HTTP-ответ, содержащий только 500-код и выводить в логи сообщение об ошибке.
### Поле needMoreReviewers у PR
При анализе не обнаружил места в API, где используется данное поле и в силу неватки времени, решил не добавлять, так как на функционал это не влияет.
